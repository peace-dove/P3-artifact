# FinBench Queries

## Overview

This directory contains query templates for different graph database systems evaluated in our experiments.

## Query Selection

Of the LDBC FinBench TCR queries, only **TCR1, TCR2, TCR5, and TCR11** involve variable-length path traversal. We use them as Q1–Q4:

| Paper | FinBench TCR |
|-------|-------------|
| Q1 | TCR1 |
| Q2 | TCR2 |
| Q3 | TCR5 |
| Q4 | TCR11 |

## Query Structure

The queries shown here are **parameterized query templates**. Each template represents the query pattern, where specific parameter values (e.g., account IDs, timestamps) are substituted at runtime from the corresponding parameter files in `params/`.

## Directory Organization

| Directory | Description |
|-----------|-------------|
| [P3](P3/) | GQL + sliding-window (SW) query templates (our system) |
| [GU](GU/) | GQL without SW — uses UDFs for temporal constraints |
| [TuGraphDB](TuGraphDB/) | Cypher query templates with TuGraphDB helper functions |
| [Neo4j](Neo4j/) | Cypher 25 query templates for Neo4j |
| [params](params/) | Batch parameter generation scripts and output files (see [params/README.md](params/README.md)) |

Queries accept batch ID inputs (`WHERE ... IN {input_ids}`). Parameter sets are generated by slicing entity `createTime` into time-based slots — 120 sets per query, 480 total. See [params/README.md](params/README.md) for details.
